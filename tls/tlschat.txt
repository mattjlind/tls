Yes — of those three, BearSSL is the one I’d look at first for an HP iPAQ 212 / Windows Mobile 6.1 project.

My ranking for your use case would be:

BearSSL — best fit

TinyTls — possible experiment, but riskier

GnuTLS — least suitable

Why:

BearSSL
BearSSL is written in C, is explicitly aimed at small embedded systems, avoids dynamic allocation, and supports TLS 1.0/1.1/1.2 with both client and server code. Its site says it targets small embedded systems, a minimal server can fit in about 20 KB of code and 25 KB of RAM, and the library depends on very little from the underlying C runtime. That combination is unusually attractive for an old ARMv5 / WinCE-class device. 

For your exact goal, BearSSL also has one big advantage over some newer embedded TLS libraries: it still supports TLS 1.2 without forcing TLS 1.3-only modern assumptions. 

The catches are that BearSSL’s published site still describes the current version as 0.6 beta-quality, and its built-in X.509 validation is described as “minimal” and does not implement revocation checks. So it looks technically promising, but you would need to test it carefully rather than assume drop-in desktop-grade behavior. 

TinyTls
TinyTls describes itself as a lightweight TLS library for embedded and resource-constrained environments, with TLS 1.2 and 1.3 support. That sounds good on paper. However, the repository appears much less mature and much less active; GitHub’s indexed page shows the latest commit surfaced there as being from about 7 years ago, with a small commit history and modest community activity. That does not make it unusable, but it makes it a riskier base for a niche WinCE port. 

There is also a practical concern for your device: TinyTls says it supports only TLS 1.2 and TLS 1.3 and explicitly says it sees no point supporting TLS 1.0. That is fine for talking to modern servers, but it suggests the project is optimized for newer assumptions rather than compatibility with older platform quirks you are likely to hit on Windows Mobile 6.1. 

GnuTLS
GnuTLS is a full-featured, mainstream TLS library with support through TLS 1.3, certificate handling, PKCS #11, TPM, DTLS, and broad protocol coverage. It also says it runs on Unix platforms and Windows. But for your case, that feature richness is more a liability than a benefit: it is a much bigger, more complex portability target than BearSSL, and nothing in the material I found suggests it is a natural fit for a WinCE/Windows Mobile 6.1 port. 

So for an iPAQ 212, GnuTLS feels too heavy, TinyTls feels too uncertain, and BearSSL feels like the best engineering bet.

The important limitation, though, is this:

None of these is likely to let you “add TLS 1.2 to the ROM” in a clean system-wide way.
What they can realistically do is let you build or patch an application so that that application speaks TLS 1.2 over raw sockets, bypassing the old Microsoft TLS stack. The hard part is not the TLS library itself; it is integrating networking, entropy, time/certificate validation, and a WinCE-compatible build environment. That is an app-level solution, not a true Schannel replacement. This follows from how these libraries are positioned and from the fact that they are standalone TLS implementations rather than drop-in Windows Mobile OS updates. 

My practical advice:

Use BearSSL if you want the best chance of getting a custom HTTPS client working on the iPAQ.

Use TinyTls only as an experiment if you are comfortable debugging an underused codebase.

Skip GnuTLS unless you end up cross-compiling in a much larger POSIX-like environment.

A sensible path would be:

build a tiny test app on the iPAQ,

open a TCP socket to a modern HTTPS endpoint,

wrap it with BearSSL client code,

hardcode one CA cert or even one server public key at first,

and only after that worry about full certificate-store integration.

Thought for 12s
Yes. Here’s a realistic BearSSL porting plan for the HP iPAQ 212 / Windows Mobile 6.1.

The key idea is: do not try to replace the OS TLS stack first. Instead, build a small standalone HTTPS client that uses BearSSL directly over Winsock. BearSSL is a compact C library intended for constrained systems, exposes its own SSL/TLS and X.509 APIs, supports TLS 1.2 client operation, and also offers a “known key” mode that can skip full certificate-chain validation when appropriate. 

What success looks like
Your first milestone should be:

open a TCP socket from WM 6.1

connect to a modern server on port 443

run a BearSSL TLS 1.2 handshake

send a simple HTTP GET

print the response body to screen or file

That is much more achievable than trying to inject TLS 1.2 into wininet.dll or the ROM. BearSSL is designed to be embedded by applications, not as a Windows Mobile Schannel replacement. 

Recommended approach
Use BearSSL in three stages:

Stage 1 — known-key / pinned-key proof of concept
Start with BearSSL’s known key model so you can avoid the hardest part first: full X.509 validation. BearSSL explicitly provides a known-key validator for cases where the client already knows the server public key out of band. That lets you prove the socket glue, record handling, and TLS handshake before worrying about CA stores and certificate parsing. 

Stage 2 — minimal CA validation
Once the proof of concept works, switch to BearSSL’s X.509 validation engine and embed a very small set of trust anchors for the servers you care about. BearSSL has its own certificate decoding and validation APIs, and its validator checks the intended server name when provided. 

Stage 3 — usable app integration
Only after the above works should you wrap it in your real application, whether that is a weather client, RSS client, or a small HTTPS fetch utility. 

Likely toolchain
The safest path is probably:

write the app in C, not C++

use a WinCE / Windows Mobile SDK toolchain you already trust

compile BearSSL as static library source directly into the project

BearSSL is plain C with very few external assumptions, which is one reason it is attractive for old embedded targets. 

Practically, I would avoid trying to port its existing build system verbatim. Instead:

create a WinCE C project

add BearSSL .c files manually

stub or adapt anything the compiler dislikes

keep the first build as small as possible

The main engineering tasks
1) TCP socket glue
BearSSL does not open sockets for you. You must supply the transport. So on WM 6.1 you will:

call WSAStartup

socket()

connect()

send() / recv()

feed bytes into BearSSL’s engine

BearSSL’s API is designed around an SSL/TLS engine plus application-managed I/O buffers, which fits this model well. 

This is the easiest part.

2) Buffer management
BearSSL expects you to provide I/O buffer space for the engine. On a memory-constrained target, you want a deliberately small configuration at first. BearSSL’s design and size calculator documentation both emphasize code-size and footprint tradeoffs depending on selected algorithms and features. 

For a first pass, keep it narrow:

TLS client only

no client certificate support

one or two cipher families only

one host only

small response bodies

3) Cipher-suite selection
On a device like the iPAQ 212, RSA verification is acceptable for a client, but you want to avoid dragging in more crypto than necessary. BearSSL supports RSA, ECDSA, ECDHE, AES/GCM, AES/CBC, ChaCha20/Poly1305 and more, with multiple implementation choices and size/performance tradeoffs. 

For first success, I would bias toward whichever common suite is easiest with the servers you control, rather than trying to support everything.

4) Certificate handling
This is one of the major pain points.

You have two practical choices:

Pinned public key first:
Use BearSSL’s known-key model and hardcode the server public key. This is ideal for a private server or your own endpoint. It avoids loading CA roots and avoids most certificate-path headaches. 

Minimal trust anchors second:
If you need general HTTPS, use BearSSL’s X.509 engine and embed a tiny set of trust anchors. BearSSL’s validator is intentionally minimal and does not implement revocation checks, so you should understand that this is not equivalent to a modern desktop trust stack. 

For your device, I would absolutely start with pinning.

5) Server-name validation
If you use full X.509 validation, make sure you pass the expected server name into the BearSSL validation flow. The BearSSL certificate-validation API explicitly uses the server name so the validator can confirm that the leaf certificate matches the intended host. 

That means:

connecting to example.com

but validating against example.com

not just “any certificate signed by this CA”

6) Time and clock validity
Old Windows Mobile devices often have bad clocks. Certificate validation depends on time being sane. If your device clock is wrong, full X.509 validation may fail even if everything else is correct.

That is another reason pinned-key mode is such a good first milestone: it sidesteps some of the practical pain while you prove the transport path. This is partly an engineering inference from how certificate validation works and from BearSSL separating known-key and X.509 validation modes. 

7) Entropy / randomness
TLS needs randomness for client nonces, ephemeral keys, and related operations. BearSSL includes PRNG interfaces, but you still need a source of seed material in your application environment. The API docs expose random-related headers and primitives, but on WinCE you will need to decide how to seed them. 

This is a real risk area.

On WM 6.1, possible seed ingredients might include:

current tick count

wall-clock time

device serial-ish values

touch timing

network timing jitter

user interaction latency

None of that is perfect. For a proof of concept, it may be enough; for serious deployment, this deserves careful thought.

What not to do first
Do not start with:

Those are all second- or third-order problems. The first question is simply: can this device complete one TLS 1.2 handshake using BearSSL in a native app?

Biggest risks
The most likely failure points are:

Compiler incompatibilities
BearSSL is portable C, but old Microsoft embedded compilers can be picky. Expect to fix:

inline usage

integer type assumptions

warnings treated as errors

possible endian or alignment assumptions

The BearSSL roadmap notes source compatibility aims before 1.0 but not necessarily binary compatibility across releases, which is another reason to compile from source and keep control of the build. 

Modern server compatibility
BearSSL supports TLS 1.2 and common algorithms, but BearSSL’s own TLS 1.3 status page says TLS 1.3 is not implemented there. That means you need servers that still negotiate TLS 1.2. Most still do, but you may need to control server-side cipher and protocol settings. 

Certificate-chain complexity
Full browser-like HTTPS behavior is the hard part, not the raw TLS handshake. Start pinned.

Performance
Handshake speed may be acceptable, but full-page browsing will still be unpleasant on that hardware. This is better suited to task-specific apps than a general web browser.

Practical development order
Here is the order I’d actually follow:

Phase A — build sanity
create WinCE console or dialog app

confirm plain HTTP works first

Phase B — BearSSL compile
add BearSSL sources

compile only what you need

Phase C — hardcoded TLS endpoint
use a server you control

Phase D — stronger validation
add hostname-aware X.509 validation

embed one or two trust anchors

test certificate expiry / hostname mismatch behavior

Phase E — applicationization
wrap in your real app

add retry logic

add better error reporting

cache downloaded content if needed

For the iPAQ 212, the best engineering path is not ROM cooking first. It is:

BearSSL in a dedicated app

pinned key first

full X.509 later if really needed

That gives you the highest chance of actually getting modern HTTPS working on the device.

A very small native example
Here is a compact starting point in C. This version uses a hardcoded IP address to keep Phase A simple.

#include <windows.h>
#include <winsock2.h>
#include <ws2tcpip.h>
#include <stdio.h>

#pragma comment(lib, "ws2.lib")

static int fetch_http(char *out, int out_size)
{
    WSADATA wsa;
    SOCKET s = INVALID_SOCKET;
    struct sockaddr_in addr;
    int ret;
    int total = 0;
    const char request[] =
        "GET / HTTP/1.0\r\n"
        "Host: 192.0.2.10\r\n"
        "\r\n";

    if (WSAStartup(MAKEWORD(2,2), &wsa) != 0) {
        return -1;
    }

    s = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    if (s == INVALID_SOCKET) {
        WSACleanup();
        return -2;
    }

    memset(&addr, 0, sizeof(addr));
    addr.sin_family = AF_INET;
    addr.sin_port = htons(80);
    addr.sin_addr.s_addr = inet_addr("192.0.2.10"); /* replace with your server IP */

    ret = connect(s, (struct sockaddr *)&addr, sizeof(addr));
    if (ret == SOCKET_ERROR) {
        closesocket(s);
        WSACleanup();
        return -3;
    }

    ret = send(s, request, (int)strlen(request), 0);
    if (ret == SOCKET_ERROR) {
        closesocket(s);
        WSACleanup();
        return -4;
    }

    while (1) {
        char buf[256];
        ret = recv(s, buf, sizeof(buf) - 1, 0);
        if (ret == 0) {
            break; /* peer closed cleanly */
        }
        if (ret == SOCKET_ERROR) {
            closesocket(s);
            WSACleanup();
            return -5;
        }

        if (total + ret < out_size - 1) {
            memcpy(out + total, buf, ret);
            total += ret;
        }
    }

    out[total] = '\0';

    closesocket(s);
    WSACleanup();
    return total;
}

while (1) {
    unsigned state = br_ssl_engine_current_state(&sc.eng);

    if (state & BR_SSL_SENDREC) {
        unsigned char *buf;
        size_t len;

        buf = br_ssl_engine_sendrec_buf(&sc.eng, &len);
        send(sock, buf, len, 0);
        br_ssl_engine_sendrec_ack(&sc.eng, len);
    }

    if (state & BR_SSL_RECVREC) {
        unsigned char buf[512];
        int r = recv(sock, buf, sizeof(buf), 0);

        if (r <= 0) break;

        unsigned char *ibuf;
        size_t ilen;

        ibuf = br_ssl_engine_recvrec_buf(&sc.eng, &ilen);
        memcpy(ibuf, buf, r);
        br_ssl_engine_recvrec_ack(&sc.eng, r);
    }

    if (state & BR_SSL_SENDAPP) {
        /* send HTTP request */
    }

    if (state & BR_SSL_RECVAPP) {
        /* read decrypted data */
    }

    if (state & BR_SSL_CLOSED) {
        break;
    }
}

static int fetch_http(char *out, int out_size)
{
    WSADATA wsa;
    SOCKET s;
    struct sockaddr_in addr;
    int ret;
    int total;
    const char *req;

    s = INVALID_SOCKET;

        "\r\n";

    if (WSAStartup(MAKEWORD(2, 2), &wsa) != 0) {
        return -1;
    }

    s = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    if (s == INVALID_SOCKET) {
        WSACleanup();
        return -2;
    }

    memset(&addr, 0, sizeof(addr));
    addr.sin_family = AF_INET;
    addr.sin_port = htons(80);
    addr.sin_addr.s_addr = inet_addr("192.168.1.100");   /* CHANGE THIS */

    if (connect(s, (struct sockaddr *)&addr, sizeof(addr)) == SOCKET_ERROR) {
        closesocket(s);
        WSACleanup();
        return -3;
    }

    ret = send(s, req, (int)strlen(req), 0);
    if (ret == SOCKET_ERROR) {
        closesocket(s);
        WSACleanup();
        return -4;
    }

    while (1) {
        char buf[256];

        ret = recv(s, buf, sizeof(buf) - 1, 0);
        if (ret == 0) {
            break;
        }
        if (ret == SOCKET_ERROR) {
            closesocket(s);
            WSACleanup();
            return -5;
        }

        if (total + ret < out_size - 1) {
            memcpy(out + total, buf, ret);
            total += ret;
        }
    }

    out[total] = '\0';

    closesocket(s);
    WSACleanup();
    return total;
}

static int tls_test(void)
{
    WSADATA wsa;
    SOCKET sock;
    struct sockaddr_in addr;

    br_ssl_client_context sc;
    br_x509_knownkey_context xc;

    unsigned char iobuf[BR_SSL_BUFSIZE_BIDI];
    unsigned char netbuf[512];

    int ret;

    /* ---- Winsock ---- */
    if (WSAStartup(MAKEWORD(2,2), &wsa) != 0) {
        MessageBox(NULL, L"WSAStartup failed", L"TLS", MB_OK);
        return 0;
    }

    sock = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    if (sock == INVALID_SOCKET) {
        MessageBox(NULL, L"Socket failed", L"TLS", MB_OK);
        return 0;
    }

    memset(&addr, 0, sizeof(addr));
    addr.sin_family = AF_INET;
    addr.sin_port = htons(4433);
    addr.sin_addr.s_addr = inet_addr("192.168.1.100"); /* CHANGE */

    if (connect(sock, (struct sockaddr*)&addr, sizeof(addr)) == SOCKET_ERROR) {
        MessageBox(NULL, L"Connect failed", L"TLS", MB_OK);
        return 0;
    }

    /* ---- BearSSL ---- */

    memset(&sc, 0, sizeof(sc));
    memset(&xc, 0, sizeof(xc));

    br_x509_knownkey_init_rsa(&xc, &SERVER_RSA_PK, 0);

    br_ssl_client_zero(&sc);

    br_ssl_engine_set_versions(&sc.eng, BR_TLS12, BR_TLS12);
    br_ssl_engine_set_buffer(&sc.eng, iobuf, sizeof(iobuf), 1);
    br_ssl_engine_set_x509(&sc.eng, &xc.vtable);

    if (!br_ssl_client_reset(&sc, NULL, 0)) {
        MessageBox(NULL, L"TLS reset failed", L"TLS", MB_OK);
        return 0;
    }

    /* ---- TLS loop ---- */

    while (1) {
        unsigned state = br_ssl_engine_current_state(&sc.eng);

        /* send TLS records */
        if (state & BR_SSL_SENDREC) {
            unsigned char *buf;
            size_t len;

            buf = br_ssl_engine_sendrec_buf(&sc.eng, &len);
            send(sock, buf, len, 0);
            br_ssl_engine_sendrec_ack(&sc.eng, len);
        }

        /* receive TLS records */
        if (state & BR_SSL_RECVREC) {
            int r;
            unsigned char *buf;
            size_t len;

            buf = br_ssl_engine_recvrec_buf(&sc.eng, &len);

            r = recv(sock, netbuf, sizeof(netbuf), 0);
            if (r <= 0) {
                MessageBox(NULL, L"Socket closed", L"TLS", MB_OK);
                break;
            }

            memcpy(buf, netbuf, r);
            br_ssl_engine_recvrec_ack(&sc.eng, r);
        }

        /* send HTTP request */
        if (state & BR_SSL_SENDAPP) {
            unsigned char *buf;
            size_t len;

            const char *req =
                "GET / HTTP/1.0\r\n"
                "\r\n";

            buf = br_ssl_engine_sendapp_buf(&sc.eng, &len);

            if (len >= strlen(req)) {
                memcpy(buf, req, strlen(req));
                br_ssl_engine_sendapp_ack(&sc.eng, strlen(req));
            }
        }

        /* receive decrypted data */
        if (state & BR_SSL_RECVAPP) {
            unsigned char *buf;
            size_t len;

            buf = br_ssl_engine_recvapp_buf(&sc.eng, &len);

            if (len > 0) {
                WCHAR wbuf[512];
                int copy = (len < 511) ? len : 511;

                MultiByteToWideChar(CP_ACP, 0, (char*)buf, copy, wbuf, 512);
                wbuf[copy] = 0;

                MessageBox(NULL, wbuf, L"TLS Response", MB_OK);

                br_ssl_engine_recvapp_ack(&sc.eng, len);
                break;
            }
        }

        if (state & BR_SSL_CLOSED) {
            MessageBox(NULL, L"TLS closed", L"TLS", MB_OK);
            break;
        }
    }

    closesocket(sock);
    WSACleanup();
    return 1;
}

static int
wince_seed(const br_prng_class **ctx)
{
    br_hmac_drbg_context *rc;
    unsigned char seed[64];
    SYSTEMTIME st;
    FILETIME ft;
    DWORD tc;
    DWORD pid;
    DWORD tid;
    int i;

    if (ctx == NULL || *ctx == NULL) {
        return 0;
    }

    rc = (br_hmac_drbg_context *)(void *)(*ctx);

    memset(seed, 0, sizeof(seed));

    GetSystemTime(&st);
    SystemTimeToFileTime(&st, &ft);
    tc = GetTickCount();
    pid = GetCurrentProcessId();
    tid = GetCurrentThreadId();

    memcpy(seed + 0, &ft.dwLowDateTime, sizeof(ft.dwLowDateTime));
    memcpy(seed + 4, &ft.dwHighDateTime, sizeof(ft.dwHighDateTime));
    memcpy(seed + 8, &tc, sizeof(tc));
    memcpy(seed + 12, &pid, sizeof(pid));
    memcpy(seed + 16, &tid, sizeof(tid));
    memcpy(seed + 20, &st, sizeof(st) < 32 ? sizeof(st) : 32);

    for (i = 32; i < 64; i++) {
        seed[i] = (unsigned char)(tc + i * 37 + (ft.dwLowDateTime >> (i & 7)));
    }

    br_hmac_drbg_update(rc, seed, sizeof(seed));
    return 1;
}

br_prng_seeder_system(const char **name)
{
    if (name != 0) {
        *name = "wince-basic";
    }
    return &wince_seed;
}

What’s really going wrong
This line in your seeder was wrong:

rc = (br_hmac_drbg_context *)(void *)(*ctx);
*ctx is the vtable pointer, not the start of the br_hmac_drbg_context object.

So later, when br_hmac_drbg_update() runs, it treats a vtable address as though it were a full DRBG context struct. That corrupts the internal pointers, and eventually you hit a datatype misalignment when BearSSL reads fields like digest_class->desc.

Fix
Change that line to:

rc = (br_hmac_drbg_context *)(void *)ctx;

#include <windows.h>
#include <winsock2.h>
#include <string.h>
#include "bearssl.h"
#include "wm_https.h"
#include "wm_cert_store.h"

#pragma comment(lib, "ws2.lib")

#define WM_HTTPS_IOBUF_SIZE BR_SSL_BUFSIZE_BIDI
#define WM_HTTPS_RECV_CHUNK 1024
#define WM_HTTPS_TIMEOUT_MS 5000

static int wm_socket_connect(const char *host, unsigned short port, SOCKET *out_sock, int *out_wsa)
{
    WSADATA wsa;
    SOCKET sock;
    struct sockaddr_in addr;

    *out_sock = INVALID_SOCKET;
    *out_wsa = 0;

    if (WSAStartup(MAKEWORD(2, 2), &wsa) != 0) {
        *out_wsa = WSAGetLastError();
        return 0;
    }

    sock = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    if (sock == INVALID_SOCKET) {
        *out_wsa = WSAGetLastError();
        WSACleanup();
        return 0;
    }

    memset(&addr, 0, sizeof(addr));
    addr.sin_family = AF_INET;
    addr.sin_port = htons(port);
    addr.sin_addr.s_addr = inet_addr(host);

    if (connect(sock, (struct sockaddr *)&addr, sizeof(addr)) == SOCKET_ERROR) {
        *out_wsa = WSAGetLastError();
        closesocket(sock);
        WSACleanup();
        return 0;
    }

    {
        int timeout_ms;
        timeout_ms = WM_HTTPS_TIMEOUT_MS;
        setsockopt(sock, SOL_SOCKET, SO_RCVTIMEO, (const char *)&timeout_ms, sizeof(timeout_ms));
    }

    *out_sock = sock;
    return 1;
}

static void wm_socket_close(SOCKET sock)
{
    if (sock != INVALID_SOCKET) {
        closesocket(sock);
    }
    WSACleanup();
}

int wm_https_get(
    const char *host,
    unsigned short port,
    const char *path,
    char *response,
    int response_size,
    wm_https_result *result
)
{
    SOCKET sock;
    br_ssl_client_context sc;
    br_x509_minimal_context xc;
    unsigned char iobuf[WM_HTTPS_IOBUF_SIZE];
    unsigned state;
    int sent_http;
    int response_len;
    int ok;
    char request[256];

    if (result != 0) {
        memset(result, 0, sizeof(*result));
    }
    if (response == 0 || response_size <= 1) {
        return 0;
    }
    response[0] = '\0';

    if (!wm_socket_connect(host, port, &sock, result ? &result->wsa_error : &(int){0})) {
        return 0;
    }

    memset(&sc, 0, sizeof(sc));
    memset(&xc, 0, sizeof(xc));
    memset(iobuf, 0, sizeof(iobuf));

    br_ssl_client_init_full(&sc, &xc, WM_TRUST_ANCHORS, WM_TRUST_ANCHOR_COUNT);
    br_ssl_engine_set_versions(&sc.eng, BR_TLS12, BR_TLS12);
    br_ssl_engine_set_buffer(&sc.eng, iobuf, sizeof(iobuf), 1);

    if (!br_ssl_client_reset(&sc, host, 0)) {
        if (result) {
            result->tls_error = br_ssl_engine_last_error(&sc.eng);
        }
        wm_socket_close(sock);
        return 0;
    }

    wsprintfA(request, "GET %s HTTP/1.0\r\nHost: %s\r\n\r\n", path, host);

    sent_http = 0;
    response_len = 0;
    ok = 0;

    while (1) {
        state = br_ssl_engine_current_state(&sc.eng);

        if (state & BR_SSL_CLOSED) {
            if (result) {
                result->tls_error = br_ssl_engine_last_error(&sc.eng);
            }
            break;
        }

        if (state & BR_SSL_SENDREC) {
            unsigned char *buf;
            size_t len;

            buf = br_ssl_engine_sendrec_buf(&sc.eng, &len);
            while (len > 0) {
                int wr;

                wr = send(sock, (const char *)buf, (int)len, 0);
                if (wr <= 0) {
                    if (result) {
                        result->wsa_error = WSAGetLastError();
                    }
                    wm_socket_close(sock);
                    return 0;
                }

                br_ssl_engine_sendrec_ack(&sc.eng, (size_t)wr);
                buf = br_ssl_engine_sendrec_buf(&sc.eng, &len);
            }
            continue;
        }

        if ((state & BR_SSL_SENDAPP) && !sent_http) {
            unsigned char *buf;
            size_t len;
            size_t qlen;

            buf = br_ssl_engine_sendapp_buf(&sc.eng, &len);
            qlen = strlen(request);

            if (len >= qlen) {
                memcpy(buf, request, qlen);
                br_ssl_engine_sendapp_ack(&sc.eng, qlen);
                br_ssl_engine_flush(&sc.eng, 0);
                sent_http = 1;
            }
            continue;
        }

        if (state & BR_SSL_RECVREC) {
            unsigned char *buf;
            size_t len;
            int rd;
            int want;

            buf = br_ssl_engine_recvrec_buf(&sc.eng, &len);
            if (len == 0) {
                break;
            }

            want = (len < WM_HTTPS_RECV_CHUNK) ? (int)len : WM_HTTPS_RECV_CHUNK;
            rd = recv(sock, (char *)buf, want, 0);
            if (rd <= 0) {
                if (result) {
                    result->wsa_error = WSAGetLastError();
                }
                wm_socket_close(sock);
                return 0;
            }

            br_ssl_engine_recvrec_ack(&sc.eng, (size_t)rd);
            continue;
        }

        if (state & BR_SSL_RECVAPP) {
            unsigned char *buf;
            size_t len;
            int take;

            buf = br_ssl_engine_recvapp_buf(&sc.eng, &len);
            if (len > 0) {
                take = (int)len;
                if (response_len + take >= response_size) {
                    take = response_size - response_len - 1;
                }

                if (take > 0) {
                    memcpy(response + response_len, buf, take);
                    response_len += take;
                    response[response_len] = '\0';
                }

                br_ssl_engine_recvapp_ack(&sc.eng, (size_t)len);
                ok = 1;
                continue;
            }
        }
    }

    wm_socket_close(sock);

    if (result) {
        result->ok = ok;
        result->http_bytes = response_len;
    }
    return ok;
}
